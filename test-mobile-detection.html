How Mobile Detection Works

This page demonstrates the mobile detection system implemented for the Ingenium website.

Device Detection Criteria:

  • User Agent Check: Detects mobile browsers (Android, iOS, etc.)
  • Screen Width: Considers devices with width ≤ 768px as mobile
  • Touch Capability: Checks for touch support

Redirect Behavior:

  • Mobile Device: Desktop pages redirect to mobile versions
  • Desktop Device: Mobile pages redirect to desktop versions
  • File Mapping: Each page has a corresponding mobile/desktop version

File Mapping

The system maps between desktop and mobile versions:

  • index.html ↔ index-mobile.html
  • about.html ↔ about-mobile.html
  • team.html ↔ team-mobile.html
  • projects.html ↔ projects-mobile.html
  • join-induvidual.html ↔ join-individual-mobile.html
  • join-club.html ↔ join-club-mobile.html
  • join-company.html ↔ join-company-mobile.html

Testing Instructions

  1. Desktop Testing: Open any mobile HTML file on desktop - it should redirect to desktop version
  2. Mobile Testing: Open any desktop HTML file on mobile - it should redirect to mobile version
  3. Browser DevTools: Use device simulation to test different screen sizes
  4. Real Devices: Test on actual mobile devices for best results

Implementation Files

  • mobile-detection.js - Detects mobile devices and redirects desktop pages
  • desktop-detection.js - Detects desktop devices and redirects mobile pages
  • Both scripts are included in all HTML files
